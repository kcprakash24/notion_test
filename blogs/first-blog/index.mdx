# Ideas behind AdaBoost



![image](./assets/Screenshot_2024-06-03_at_8.58.42_AM.png)



A tree with just 2 nodes is called a **stump**. Stumps are not great at making accurate classifications. Stumps are weak learners.

1. AdaBoost combines lot of weak learners for classification and they almost always use stumps

1. In random forest, we take vote of decision trees to get the final label for classification, but in AdaBoost that’s not the case, some stumps have more say in classification than others.

1. In random forest, it doesn’t matter if the trees are created in some order because ultimately we use voting to get the final output. But in AdaBoost, each stump is made by taking the previous stump’s mistake into account.

### Step 4 - Creation of bucket column

Let us now create a second stump

After preparing the normalized weight, we prepare one more column in data set that has buckets column that has some range of numbers derived from **sample weight column.**

how each value is calculated in the bucket column

```plain text
value = [start, end]
start = (n-1)th row of sample weight
end = (n-1)th row of bucket + nth row of sample weight
```



We now prepare new dataset of same size, but without sample weight and buckets columns. 

We prepare the dataset by selecting random numbers between 0, 1 and wherever this random number falls in the bucket value range, we choose that row and add it in the new dataset. We run this iteration until the new dataset is of same length of the original one.

### How AdaBoost Classifies

![image](./assets/Untitled.png)

Look at the above figure, assume, the left stumps in the forest classifies Has Heart Disease and right stumps classify Does not have heart disease.

We calculate Amount of Say of all the stumps in left and right. We then sum the Amount of Say of left and right stumps.

Whichever sum is highest, we use that label as our output which in this case is left stumps and classified output will be Has Heart Disease.
